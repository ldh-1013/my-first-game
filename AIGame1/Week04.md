# 4주차 실습 정리

## 오늘 한 것
- 원형 / AABB / OBB Bounding Box 시각화
- 세 방식의 충돌 판정 차이 관찰

## 관찰 내용

### 원형 vs AABB
AABB (빨간색): 사각형의 모서리 끝까지 충돌 영역으로 간주한다.
따라서 시각적으로는 닿지 않은 것 같아도 빨간 테두리끼리 스치면 충돌로 판정될 수 있다.

원형 (초록색): 모서리 부분에 빈 공간이 생긴다.
사각형의 모서리가 상대방 사각형 안으로 깊숙이 들어가더라도, 초록색 원끼리 닿지 않으면 배경색이 노란색으로 변하지 않는다는 것을
관찰 할 수 있다.

### AABB vs OBB

AABB (빨간색): 오브젝트가 회전하면 이를 감싸기 위해 사각형의 크기가 변한다.
특히 사각형이 45도 기울어졌을 때 AABB가 가장 커지는데
이때 실제 물체가 없는 모서리 근처의 빈 공간에 플레이어가 닿아도 "HIT" 판정이 나게 된다.

OBB (초록색): 오브젝트가 회전하더라도 경계 상자의 크기는 변하지 않고 물체와 함께 회전한다.
물체의 실제 외곽선에 딱 붙어 있으므로, AABB에서 발생하던 빈 공간 문제가 사라지게된다.

## AI와의 대화에서 배운 것

AABB와 OBB의 결정적인 차이와 수학적인 차이까지 배우게 되었으며
AABB와 OBB의 장단점 및 용도에 대해서 ai의 도움을 받아 직접 실습해보며 배웠다.

## 내 게임에 적용한다면

처음엔 대충 훑는 느낌으로 AABB를 사용하겠지만, 최종적으로는 OBB를 쓸 것이다.
이유는 AABB보다 OBB는 성능과 정밀도의 밸런스가 좋고 무엇보다 OBB가 매우 정밀하여 사용하기에 용이할 것 같다.
